# 03_image — Pinterest pin image generation prompt

Each article needs a **1000x1500 px portrait image** for Pinterest. Pinterest algorithm rewards aspect ratio 2:3 specifically.

## Two routes

### Route A — Programmatic (default, free)

Generated by `site/scripts/generate-og.ts` using `@vercel/og` or Satori at build time:

- Read article title from `messages/en.json` (and per locale)
- Render JSX template with brand colors, title, category badge, year
- Output to `site/public/og/<slug>-<locale>.png`

This is the default. No external API needed. Works for all 17 locales × N articles.

Template requirements:
- Background: gradient or solid brand color
- Title: 36-48px, bold, max 8 words, ellipsis if longer
- Category chip: small pill at top
- Site name: bottom right, small
- Year: top right (e.g. "2026")
- High contrast for mobile feed visibility

### Route B — AI-generated visual (optional, paid)

When article warrants a custom hero image (visual product comparison, lifestyle shot):

Use prompt template:

```
A clean, minimalist Pinterest-style infographic showing {topic}. Vertical 2:3 aspect ratio. Bright but not neon colors. No text rendering required (text added programmatically). Pinterest aesthetic: airy, scannable, product-focused. No watermark.
```

External tools (user-provided API keys, optional):
- Gemini 2.x Image / Nano Banana
- DALL-E 3 / 4
- Imagen 4
- Midjourney v7

Save raw image to `site/public/og/_raw/<slug>.png`, then overlay text via build step → `site/public/og/<slug>-<locale>.png`.

## Pin description (separate from image)

Already generated by `01_write` and `02_translate` as `pinDescription` field. Pinterest API takes this as the `description` field of the pin object.

## Pin link

Always: `https://<NEXT_PUBLIC_SITE_URL>/<locale>/articles/<slug>/`

## Quality checks

- [ ] Image is exactly 1000x1500
- [ ] Title fits (no overflow, no orphan word)
- [ ] Category badge present
- [ ] Brand mark present
- [ ] File size <500KB (Pinterest guideline)
